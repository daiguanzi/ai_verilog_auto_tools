# 饮料售货机 FSM 仿真报告

## 设计概述

- **模块**: `vending_machine`
- **功能**: 3 种硬币 (0.5 / 1 / 2 元)，饮料售价 1.5 元
- **状态**: ACCUMULATE (累积) → DISPENSE (出货) / REFUND (退款)
- **金额单位**: 0.1 元 (1.5 元 = 15 单位)

### 接口

| 信号 | 方向 | 宽度 | 说明 |
|------|------|------|------|
| `clk` | input | 1 | 时钟 |
| `rst_n` | input | 1 | 异步复位 (低有效) |
| `coin_05` | input | 1 | 投入 0.5 元 (脉冲) |
| `coin_1` | input | 1 | 投入 1 元 (脉冲) |
| `coin_2` | input | 1 | 投入 2 元 (脉冲) |
| `cancel` | input | 1 | 退钱请求 (金额 < 1.5 时有效) |
| `amount` | output | 5 | 当前已投金额 (0.1 元单位) |
| `dispense` | output | 1 | 出货脉冲 (1 周期) |
| `change` | output | 5 | 找零金额 (0.1 元单位) |
| `change_valid` | output | 1 | 找零有效脉冲 (1 周期) |

## 仿真测试结果 (9/9 通过)

### 1. 正好支付 (exact_payment)
**投入 0.5 × 3 = 1.5 元，正好支付，无找零**

| 时间 (ns) | 事件 | amount | dispense | change_valid | change |
|-----------|------|--------|----------|-------------|--------|
| 40 | 初始化 | 0 (0.0) | 0 | 0 | 0 (0.0) |
| 70 | 投入第 1 个 0.5 元 | 5 (0.5) | 0 | 0 | 0 (0.0) |
| 100 | 投入第 2 个 0.5 元 (累计 1.0) | 10 (1.0) | 0 | 0 | 0 (0.0) |
| 130 | 投入第 3 个 0.5 元 (累计 1.5) | 0 (0.0) | **1** | 0 | 0 (0.0) |

> 投入完 3 枚 0.5 元后，amount 达到 15 (1.5 元)，dispense 信号置 1，出货饮料；金额归零，无找零。

## FSM 状态转换图

```
              coin × N
    ┌─────────────────────────┐
    │                         │
    ▼                         │
┌──────────┐  amount<15, cancel  ┌─────────┐
│ACCUMULATE│ ──────────────────► │ REFUND  │──┐
└──────────┘                     └─────────┘  │
    │                                         │
    │ amount >= 15                            │
    ▼                                         │
┌──────────┐                                  │
│ DISPENSE │──────────────────────────────────┘
└──────────┘   (自动返回 ACCUMULATE)
```

---

## 关键实现细节

1. **金额累加**: 使用 `amount_d = amount_d + N` 而非 `amount_d = amount_q + N`，确保同时投多币时正确累加
2. **脉冲信号**: dispense 和 change_valid 为寄存器输出，每次恰好持续 1 个时钟周期
3. **状态编码**: 2 位状态机 (S_ACCUMULATE=00, S_DISPENSE=01, S_REFUND=10)
4. **金额范围**: amount 最大 20 (2.0 元)，5 位宽度足够
